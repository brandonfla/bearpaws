# Bearpaws v2.0.0 — independent line

This release marks Bearpaws' clean break from the upstream superpowers line. It is a positioning bump, not a behavioral break: the skill set, bootstrap mechanism, and install flows are unchanged from 1.1.1.

## Why 2.0.0

The 1.x series was framed as a hard fork of [superpowers](https://github.com/obra/superpowers) v5.0.7. With the public positioning settled (1.1.1) and the project evolving on its own cadence, continuing to track upstream version semantics no longer reflects how Bearpaws is developed or supported. 2.0.0 starts a Bearpaws-native version line.

## What changed

- Plugin version bumped from 1.1.1 to 2.0.0 across `package.json`, `.claude-plugin/plugin.json`, `.claude-plugin/marketplace.json`, and `gemini-extension.json`.

## What did not change

- No skill bodies, no bootstrap content, no validator gate.
- No install control flow, hook layout, or symlink topology.
- No support-tier promotions: Claude Code and Gemini CLI remain primary; Codex, Devin for Terminal, and Windsurf Cascade remain experimental.
- Attribution to Jesse Vincent and superpowers contributors and MIT license continuity are preserved unchanged.
