# Bearpaws v1.0.0 — Release

**Released:** 2026-05-01

First stable release. All four phases of the fork program are complete: identity (Phase 0), vertical slice (Phase 1), full migration (Phase 2), and integration (Phase 3).

## What ships in 1.0.0

- **24 skills** (14 process + 10 domain) — all in XML schema, zero schema violations.
- **10 domain skills** covering Google Cloud, Google ADK, Vite, JS/TypeScript, and Cloud Run — each as a reference + workflow pair.
- **50.7% byte reduction** on process skills (108,393 → 53,477 bytes) via XML schema migration, `_shared/` extraction, and content condensation.
- **45% bootstrap reduction** (5,292 → 2,907 bytes injected per session).
- **`<see>` lazy-load convention** — heavy reference files (>150 lines) are pointer-only by default; agents load them only when explicitly relevant.
- **Schema validator** scoped to SKILL.md and `_shared/` files — supporting files (HTML templates, external docs, code examples) are excluded.
- **Code-reviewer agent** updated with domain-specific review criteria for GCP/Cloud Run, ADK, Vite, and TypeScript.

## Phase 3 changes (since v0.2.0 / Phase 2)

- Verified all >150-line supporting files are properly `<see>`-demoted or excluded from agent context.
- Schema validator scope corrected: validates SKILL.md + `_shared/*.md` only, not HTML templates or external reference docs.
- Code-reviewer agent (`agents/code-reviewer.md`) gains a "Domain-Specific Review" section covering IAM/secrets/scaling (GCP), agent shapes/tools (ADK), plugin ordering/env conventions (Vite), and strict-mode/type-narrowing (TypeScript).
- Version bumped from 0.2.0 → 1.0.0 across all 4 manifests.
- README updated with final measurements, full skill list, and current install instructions.

## Known limitations

- **Gemini CLI compatibility** not smoke-tested in this release (requires Gemini CLI installation). The `gemini-extension.json` manifest and `using-bearpaws` bootstrap include Gemini-specific notes, but end-to-end triggering is unvalidated since v0.1.0.
- **`<include>` is advisory**, not imperative. Agents treat `<include ref="..."/>` as annotation; they do not auto-read the referenced file. Documented in `writing-skills` SKILL.md. The single `<include>` usage (`_shared/red-flags-skill-discipline` in TDD) contains non-critical meta-cognitive content.
- **Generic prompts bypass skill checks.** Prompts like "How do I rename a function?" do not trigger skills — this is a pre-existing bootstrap gap, not a regression.
