# Bearpaws v1.0.0 — public baseline

First public release of Bearpaws. This doc establishes the v1 baseline; subsequent release notes in this directory record what changes from here.

## What Bearpaws is

A skills plugin for Claude Code, Gemini CLI, Devin for Terminal, and Windsurf Cascade. v1 ships 15 stack-agnostic skills covering TDD, debugging, planning, code review, and parallel execution, plus a bootstrap skill that establishes skill-discovery discipline at session start.

## Goals

- **Mitigate token usage and enforce token efficiency.** Keep behavioral parity with the upstream skills library while shrinking the per-session context footprint as far as the wording will tolerate.
- **Stack-agnostic.** No framework or cloud-vendor opinions baked into the skill set; project-specific conventions are discovered via onboarding rather than imposed by domain skills.
- **Same discipline across harnesses.** One bootstrap, one skill format, one set of behavioral rules — Claude Code, Gemini CLI, Devin, and Windsurf all read from the same `skills/` tree.

## What's included in v1

**Bootstrap (1):** `using-bearpaws`

**Always-first (1):** `onboarding-to-a-project`

**Process skills (13):** `brainstorming`, `writing-plans`, `executing-plans`, `test-driven-development`, `systematic-debugging`, `verification-before-completion`, `requesting-code-review`, `receiving-code-review`, `finishing-a-development-branch`, `subagent-driven-development`, `dispatching-parallel-agents`, `using-git-worktrees`, `writing-skills`

It is not affiliated with or endorsed by the superpowers project.

The fork exists to pursue the token-efficiency goal above: same behavioral rules, expressed more compactly, with deferred loading and a structured XML schema.

## Token efficiency at v1

Approximations from a single point-in-time measurement against superpowers v5.0.7. Treat as direction, not commitments:

- Bootstrap injected per session: ~4.3 KB (~1.0K tokens), roughly 20% smaller than upstream.
- Process skill bodies (apples-to-apples subset): roughly half the byte count of upstream.

Token counts measured with `tiktoken` `cl100k_base` as a proxy for Anthropic's tokenizer. These numbers will drift as either project evolves.
