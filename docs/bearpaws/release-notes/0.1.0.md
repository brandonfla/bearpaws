## Test infrastructure fixes shipped in 0.1.0

The upstream test scripts assumed a Linux-shaped environment. Phase 0 added portability fixes so the suite runs on macOS and inside an active Claude Code session:

- `tests/skill-triggering/run-test.sh` and `tests/explicit-skill-requests/run-test.sh`:
  - Use `gtimeout` (macOS coreutils) when GNU `timeout` is unavailable; fall through to no-timeout when neither is present.
  - `env -u CLAUDECODE` before invoking `claude -p` so the suite can run from inside a Claude Code session without nested-session refusal.
  - Append `--verbose` to the `claude -p` invocation (newer `claude` CLI requires it with `--output-format stream-json`).
- `tests/skill-triggering/run-all.sh`: read `${PIPESTATUS[0]}` instead of trusting the exit status of `tee`, so failures are surfaced in the summary.

## Changes from superpowers v5.0.7

- Plugin slug: `superpowers` → `bearpaws`.
- Marketplace: `superpowers-dev` → `bearpaws-dev`.
- Dropped platforms: Cursor, OpenAI Codex (manifests, install docs, reference docs).
- Dropped reference docs: `skills/using-superpowers/references/copilot-tools.md`, `skills/using-superpowers/references/codex-tools.md`.
- Renamed skill: `using-superpowers/` → `using-bearpaws/`. Skill body unchanged except for plugin-name swaps and removal of the platform-adaptation paragraph naming the dropped platforms.
- Hook namespace: `superpowers` → `bearpaws` (script comments, legacy-config path, wrapper text).
- Slash commands: `superpowers:brainstorming|executing-plans|writing-plans` → `bearpaws:*`.
- Skill cross-references: every `superpowers:<skill>` body reference rewritten to `bearpaws:<skill>` (systematic-debugging, executing-plans, requesting-code-review, writing-plans, writing-skills, subagent-driven-development).
- Manifest count: 6 → 4 (`.cursor-plugin/plugin.json`, `.codex-plugin/plugin.json` removed; `package.json`, `.claude-plugin/plugin.json`, `.claude-plugin/marketplace.json[plugins[0]]`, `gemini-extension.json` retained).
- `.version-bump.json` updated to match (cursor/codex entries removed, audit excludes added for bearpaws spec/plan docs).
- `CLAUDE.md` and `README.md` rewritten with bearpaws identity and superpowers attribution.
- `.github/PULL_REQUEST_TEMPLATE.md`, `.github/ISSUE_TEMPLATE/*.md`, `tests/opencode/*.sh`, `tests/subagent-driven-dev/*`, `docs/testing.md` updated to reference Bearpaws.

No skill body content (Red Flags tables, rationalization lists, behavioral rules) was changed — only plugin-identity strings.
