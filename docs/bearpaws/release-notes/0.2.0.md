# Bearpaws v0.2.0 — Phase 1 (Vertical Slice)

**Released:** 2026-05-01

Phase 1 proves the XML schema, `_shared/` library, lazy-load convention, and eval workflow on a controlled vertical slice. All four exit criteria pass.

## Deliverables

### XML schema migration
- **Bootstrap (`using-bearpaws`)** migrated to XML schema: `<skill>`, `<subagent-stop>`, `<warning level="hard">`, `<triggers>`/`<rule>`, `<process>`/`<step>`, `<see>`. Red Flags table compressed from 12 to 8 inline rows; 4 meta-cognitive rows extracted to `skills/_shared/red-flags-skill-discipline.md`.
- **TDD (`test-driven-development`)** migrated to XML schema: same tag set plus `<flow>`, `<example>`, `<antipattern>`, `<include>`, `<gate>`.
- **SessionStart hook** wrapper changed from `<EXTREMELY_IMPORTANT>` to `<warning level="hard">`.
- **Tag whitelist** documented in `skills/writing-skills/SKILL.md` (appended "## XML schema" section).

### Domain skills
- **`cloud-run`** (reference skill): 9 rules covering services vs. jobs, revision/traffic model, scaling, concurrency, identity, auth, secrets, VPC.
- **`deploying-to-cloud-run`** (workflow skill): pre-deploy checklist gate + deploy command + post-deploy verification.

### Tooling
- **Schema validator** (`tests/schema-validator/run-validator.sh`): grep-based enforcement of the tag whitelist. Currently 111 violations from unmigrated skills; drops to zero as Phase 2 migrations land.
- **Token measurement script** (`tests/token-measurement/measure.sh`): deterministic byte counter for bootstrap and skills tree. JSON output for diff-friendly tracking.

## Findings

### `<include>` does not auto-resolve (R2)
The `<include ref="_shared/red-flags-skill-discipline"/>` tag in the TDD skill is NOT automatically read by the agent when the skill is invoked. The agent treats the tag as annotation, not as an action to perform a `Read` call. The included content (4 meta-cognitive red flags rows) is non-critical for TDD behavior, so the tag is kept for structural dedup. Phase 3 should investigate either teaching the agent to resolve `<include>` tags or formally downgrading `<include>` to `<see>` semantics.

### Generic pressure prompts are a pre-existing gap
Prompts like "How do I rename a function?" and "What's the difference between let and const?" bypass skill checks with both the old `<EXTREMELY_IMPORTANT>` and new `<warning level="hard">` wrappers. This is a bootstrap persuasiveness gap, not a schema regression. Domain-specific prompts (those that match a skill's `description` field) reliably trigger.

### Gemini CLI smoke test deferred
Step 5.7 (Gemini CLI compatibility) was not run — requires Gemini CLI installation. Carried into Phase 2 as a validation item.
