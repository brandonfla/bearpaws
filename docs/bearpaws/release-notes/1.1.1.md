# Bearpaws — independent positioning

This release updates Bearpaws' public positioning and support documentation. It does not change source skill behavior.

## What changed

- Reframes Bearpaws as an independent, low-token skills toolkit that began as a hard fork of superpowers v5.0.7 and now evolves independently.
- Preserves attribution to Jesse Vincent and superpowers contributors, plus MIT license continuity.
- Documents Claude Code and Gemini CLI as primary supported targets.
- Documents Codex, Devin for Terminal, and Windsurf Cascade as experimental unless a specific workflow has been validated.
- Adds a descriptive skill-structure guide so the current `SKILL.md` layout is documented without introducing a heavier formal schema.
- Adds an agent-support guide that records current evidence, limitations, and promotion rules by target.
- Softens install and manifest wording where previous language implied broader support or parity than the repo currently proves.

## What did not change

- No source skill bodies were rewritten.
- No schema or validator gate was expanded.
- No generated adapter pipeline was introduced.
- No install control flow changed.
- No experimental agent was promoted to primary support.

## Where to look

- `README.md` — current public positioning and install notes.
- `docs/skill-structure.md` — descriptive contract for the current skill layout.
- `docs/agent-support.md` — support tiers, evidence, and limitations.
- `docs/migration-audit.md` — historical point-in-time audit that justified the lightweight documentation path.
